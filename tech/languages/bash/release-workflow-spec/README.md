# Bash Release-Workflow Spec

The **bash binding** of the general release-workflow principles. This shelf is the canonical
reference for releasing and distributing a pure-Bash CLI on Linux: versioning, changelog, the
GNU-conventions Makefile, the tag-triggered CI release job, and the end-user distribution channels
(`install.sh`, AUR, OBS).

For the language-agnostic principles these chapters apply, see
**[general principles](../../../programming/release-workflow/README.md)**. Every chapter here maps
back to its general counterpart (see the "General principle" column below).

## Bash diverges from the registry-publish model

The general model assumes a central package registry (crates.io, npm, PyPI) where "publish" pushes
an artifact to a registry that _is_ the release. **Bash has no such registry.** A Bash program's
"release" is:

- a signed `v*` **git tag** (the single source of truth), plus
- a **GitHub/GitLab Release** (tarball + `SHA256SUMS` + provenance), plus
- **distribution channels** that key off that tag: `install.sh`, AUR, OBS, distro packages.

**Tagging is publishing.** There is nothing to `cargo publish` / `npm publish`; the tag fans out to
every channel instead.

## TL;DR

- One source of truth: a signed `v*` git tag. **Tag → tarball → GitHub Release → `install.sh` +
  AUR + OBS.** No central registry — tagging is publishing.
- `VERSION` file at repo root; SemVer; `version` subcommand reads it (or a `__TOOL_VERSION__`
  placeholder), with `git describe` fallback in dev checkouts.
- Conventional Commits enforced by `commitlint`; `CHANGELOG.md` generated by `git-cliff`
  (`cliff.toml`, Keep-a-Changelog preset).
- Makefile follows GNU Directory Variables + `DESTDIR`; every install path is
  `$(DESTDIR)$(prefix-var)`; `dist` uses `git archive` + `sha256sum`.
- `.github/workflows/release.yml` triggers on `v*`: test → `make dist` → git-cliff notes →
  `actions/attest-build-provenance` (SLSA L3, keyless) → `gh release create` → one-line `curl` to
  trigger an OBS service run.
- Distribution channels, in priority order: `install.sh` (`curl | bash`, checksum-verified) → AUR
  (`<tool>` + `<tool>-git`) → OBS-hosted `zypper`/`dnf`/`apt` repos.
- Working branches are `develop`/`master`; the release tag is cut on `master`.
